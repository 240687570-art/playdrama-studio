# PlayDrama 客户 3 分钟演示路径

目标：让客户在三分钟内看懂 PlayDrama 不是普通短剧编辑器，而是从生成、互动制作、发布、收款到数据回流的一套商用工作台。

## 演示前准备

- 打开线上工作台：`https://playdrama.tokenaicloud.com/studio`
- 使用手机号验证码登录。
- 选择一个已有样片或新建作品，建议用悬疑反转样片。
- 支付通道当前只演示微信支付；支付宝协议到期前不要展示支付宝收银台。

## 3 分钟讲解线

### 0:00 - 0:30 生成短剧生产包

进入 `AI 导演`，说明普通短剧生成是第二业务和生产入口：

- 填题材钩子、类型、平台、时长和变现方式。
- 点击 `生成短剧生产包`。
- 展示生成后交付能力：生成后细修、生产导出、互动升级、视频回填。

客户要听到的重点：普通短剧不是孤立生成，结果会进入后续可编辑、可交付、可升级的生产台。

### 0:30 - 1:20 剧情编辑商业制作台

进入 `剧情编辑`：

- 展示剧情图谱、问题队列、主线验收、付费路径。
- 点击 `生成付费线`，说明系统会保留免费结局，并补出付费隐藏结局和解锁条件。
- 展示 `主路径 / 付费路径对比` 和 `节点收益预估`。
- 点击某个付费结局，展示节点权限可设为免费或付费。

客户要听到的重点：这里不是普通节点编辑器，而是互动短剧的商业结构设计台。

### 1:20 - 2:20 发布变现验收

进入 `发布变现`：

- 展示发布与收入门禁：发布包、内容安全、支付通道、渠道任务。
- 展示变现策略：价格档位、试看权益、付费卡点、千次会话收入预估。
- 展示发布前风险提示：内容、价格、付费路径。
- 展示支付异常处理：微信支付提示、支付失败重试、支付宝协议到期隐藏、未支付不解锁。

客户要听到的重点：平台不是只生成页面，而是把上线前风险和收款异常都放在运营台里。

### 2:20 - 3:00 真实收款和数据回流

演示完整闭环：

- 手机号登录。
- 保存作品。
- 生成发布包。
- 打开预览。
- 进入付费结局。
- 微信 0.01 元真实支付。
- 回到页面刷新订单，确认付费结局解锁。
- 展示订单明细、收入、渠道漏斗和互动数据。

客户要听到的重点：这条链路能从内容生成走到真实收款，再回到数据看板。

## 演示红线

- 支付宝协议未恢复前，不承诺支付宝可用。
- 未完成真实扫码前，不说“今天已完成真实扣款”，只能说“微信真实小额支付曾在 2026-05-29 验收通过，当前可重新跑一次”。
- 不展示任何 API Key、商户私钥、服务器 env。
- 不把本地演示态当成线上商用态，线上以 `readiness 17/17 pass` 和 provider smoke 为准。

## 客户追问话术

- 问“大厂短剧生成差在哪”：PlayDrama 主业务是互动短剧商业闭环，普通短剧生成作为生产入口，重点是生成后可编辑、可发布、可收款、可看数据。
- 问“能不能接大厂 API”：可以接，平台已有模型路由和视频队列接口，当前文本生成走 Qwen，视频生产按服务商配置进入真实生成或 Prompt 队列。
- 问“能不能马上试用”：可以给灰度试用链接，但支付链路建议现场跑一次微信 0.01 元，确认客户手机端体验。
