# 下一步计划

## 当前阶段

PlayDrama 已进入内测发布准备阶段。生产链路已经具备：阿里云 ECS、RDS PostgreSQL、腾讯云短信登录、阿里云邮件、通义千问、内容安全、本地发布页、埋点、版本历史、支付宝真实下单链路。

## 已完成

- 云数据库：RDS PostgreSQL 已接入并通过结构校验。
- Auth：腾讯云短信验证码登录已接入，页面会展示“手机号验证码登录”。
- 邮件：阿里云邮件推送已接入，用于邀请邮件。
- AI：通义千问已接入，AI 生成和用量记录可用。
- 内容安全：发布前会扫描标题、节点、选项、角色和变量文本。
- 发布：H5 发布页、发布包历史、发布预览、运行时埋点已接入。
- 支付：支付宝真实下单链路已接入，未支付订单不会提前解锁付费结局。
- 运维：阿里云脚本支持 `status`、`health`、`payment`、`smoke`、`backup`、`cleanup-smoke`。
- 预热物料：已完成项目卖点、社群文案、朋友圈文案、内测招募文案和首张宣传海报。

## 不需要用户配合的收尾项

- 持续做 UI 文案和状态打磨，避免显示内部工程词。
- 把支付、商用门禁、线上烟测放进部署验收流程。
- 每次上线后运行构建、Lint、生产健康检查、支付下单验收和线上烟测。
- 更新发布包，避免使用过期 zip。
- 把营销物料按“海报、社群短文案、内测招募、官网预热段落”沉淀到项目文档。

## 需要用户配合的事项

- 微信支付：补商户私钥 PEM 和平台公钥 PEM，之后才能把 `PAYMENT_PROVIDER` 扩展为 `alipay,wechat`。
- 域名：`playdrama.cn` 完成备案和解析后，再把 `APP_BASE_URL`、`PLAYDRAMA_API_BASE` 和支付回调 URL 切到正式域名。
- 支付实测：用真实支付宝做一笔小额支付，确认商户后台异步通知、订单解锁和对账记录。
- 法务文本：隐私政策、用户协议、付费说明、内容合规说明。
- 内容运营：准备 3 个可公开演示的互动短剧样例。

## 推荐下午顺序

1. 做支付宝小额实付回调验收。
2. 补微信支付 PEM 文件并启用微信支付。
3. 确认 `playdrama.cn` 备案进度和解析计划。
4. 写首批内测创作者邀请名单和反馈表。
